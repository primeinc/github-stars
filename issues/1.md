### Title
Fix 03-Classify Repos failures by refactoring taxonomy enforcement into TS code + executable repro

### Body
Failing job:
- https://github.com/primeinc/github-stars/actions/runs/22339929257/job/64640771254

**Problem**
The 03-Classify Repos workflow fails during Post-Classification Verification because repos.yml contains categories/frameworks that are not present in the embedded taxonomy allow-list (e.g. infrastructure, cli-tools). The workflow currently sanitizes AI output only for repos in the current batch, but verification validates the entire manifest, so any legacy invalid values anywhere hard-fail the run.

**Goal**
Refactor taxonomy enforcement into reusable TypeScript code and make the behavior provable via both automated tests and an executable reproduction command.

**Non-negotiable constraints**
- Keep taxonomy validation strict; do not relax gates to make CI green.
- Implement normalization + validation in real code (TypeScript).
- Use YAML fixtures for repro.
- The solution must include a runnable command that shows failure-before and success-after (not tests-only).
- Security: taxonomy allow-lists are authoritative; treat AI output as untrusted.

**Scope**
1) Refactor into TS code
- Add modules (suggested: src/manifest/*): load, normalize, validate, write.
- Canonicalization:
  - categories: trim + lowercase, filter against taxonomy allow-list, preserve unclassified as fallback; if empty after filtering => ["unclassified"].
  - framework: if not in allow-list => null.
  - if normalization changes anything: set needs_review: true.

2) Executable repro (required)
Add pnpm scripts (names flexible):
- pnpm repro:taxonomy
  - Phase BEFORE: strict validate fixtures/repos.invalid.yml and exit non-zero with clear error list.
  - Phase AFTER: normalize the same fixture in-memory (or to a temp file), then strict validate and exit 0.
  - Print a concise summary: repos modified count, needs_review count, bounded list of changed repos.

3) Wire into workflow
Update .github/workflows/03-classify-repos.yml to call the TS scripts instead of embedding the taxonomy enforcement in actions/github-script. Keep strict validation in CI.

4) Tests (required)
Add unit tests covering:
- all-invalid categories => unclassified + needs_review
- mixed valid/invalid => invalid removed
- invalid framework => null + needs_review
- casing/whitespace normalization

**Acceptance Criteria**
- pnpm repro:taxonomy demonstrates fail->pass behavior deterministically.
- Workflow no longer fails due to legacy invalid categories/frameworks.
- After normalization, strict validation over the full manifest yields 0 taxonomy violations.
- Logs are auditable and bounded.

**Test Plan**
Local:
- pnpm install
- pnpm repro:taxonomy
- pnpm test
CI:
- ensure pnpm repro:taxonomy and pnpm test run in workflow/CI.

**Risks**
- Large diffs if many repos drifted; provide --check/dry-run modes and bounded logging.
- False invalidation via casing; canonicalize both taxonomy and repo fields.

**Definition of Done**
- TS code is the single source of truth for normalize+validate.
- Executable repro command exists and is required.
- Tests added.
- The failing job URL no longer fails for taxonomy drift.
